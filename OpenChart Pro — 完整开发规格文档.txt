## 一、项目概述

### 1.1 项目定位
构建一个类似 TradingView 的专业级**自托管看盘平台**，命名为 **OpenChart Pro**。支持加密货币、美股、港股、A股四大市场，具备实时K线图表、20+内置技术指标、自定义公式编辑器、智能警报、策略回测、AI智能选股和加密货币链上数据仪表盘。

### 1.2 技术栈总览

| 层级 | 技术选型 | 版本要求 |
|------|---------|---------|
| 后端框架 | FastAPI | >=0.100 |
| 实时通信 | WebSocket (FastAPI 原生) | - |
| K线图表 | KLineChart Pro | v10+ |
| 代码编辑器 | CodeMirror 6 | v6+ |
| 后端计算 | NumPy + Pandas | >=1.24 / >=2.0 |
| 回测引擎 | VectorBT | >=0.26 |
| AI/LLM | DeepSeek API / 通义千问 API (可配置) | - |
| 数据存储 | SQLite | 3.x |
| 任务调度 | APScheduler | >=3.10 |
| HTTP客户端 | aiohttp | >=3.9 |
| 前端样式 | 原生CSS (暗色主题) | - |
| 包管理 | pip + npm | - |

### 2.2 数据模型定义 — data/models.py

```python
"""
统一数据模型，所有数据源输出都转换为这些标准格式。
"""
from dataclasses import dataclass, field
from typing import List, Optional, Dict, Any
from enum import Enum

class Market(Enum):
    CRYPTO = "crypto"
    US = "us"
    HK = "hk"
    CN = "cn"

class Interval(Enum):
    M1 = "1m"
    M5 = "5m"
    M15 = "15m"
    M30 = "30m"
    H1 = "1H"
    H4 = "4H"
    D1 = "1D"
    W1 = "1W"
    MN = "1M"

@dataclass
class Candle:
    timestamp: int          # 毫秒时间戳
    open: float
    high: float
    low: float
    close: float
    volume: float
    turnover: float = 0.0  # 成交额（可选）

@dataclass
class Symbol:
    symbol: str             # 如 "BTC-USDT", "AAPL", "0700.HK", "600519.SH"
    name: str               # 显示名称
    market: Market
    exchange: str           # 交易所/数据源名称
    base: str = ""          # 基础货币（加密用）
    quote: str = ""         # 计价货币（加密用）

@dataclass
class KlineData:
    symbol: str
    market: Market
    interval: Interval
    candles: List[Candle]
    indicators: Dict[str, List[float]] = field(default_factory=dict)

@dataclass
class Alert:
    id: str
    symbol: str
    market: Market
    condition_type: str     # "price" | "indicator" | "crossover" | "volume" | "change" | "formula"
    condition: Dict[str, Any]
    message: str
    notify_methods: List[str] = field(default_factory=lambda: ["browser", "sound"])  # "browser" | "sound" | "webhook"
    label: str = ""         # 备注标签（如"突破关键压力位"）
    repeat_mode: str        # "once" | "every_time"
    cooldown: int = 300     # 秒
    enabled: bool = True
    triggered_count: int = 0
    last_triggered: Optional[int] = None

@dataclass
class BacktestResult:
    """
    回测结果数据对象。
    ⚠️ 此 dataclass 是 2.7.2 report JSON 的 Python 表示。
    API 返回值以 2.7.2 的 JSON 格式为准（含 summary 嵌套对象）。
    """
    id: str
    strategy_name: str
    symbol: str
    interval: str
    start_date: str
    end_date: str
    summary: Dict[str, Any]     # 核心指标（见 2.7.2 report JSON 的 summary 对象）
    equity_curve: List[Dict]    # 资金曲线 [{"time": ts, "value": float}]
    benchmark_curve: List[Dict] # 基准曲线（买入持有）
    drawdown_curve: List[Dict]  # 回撤曲线
    trades: List[Dict]          # 交易记录明细
    monthly_returns: Dict       # 月度收益
    optimization: Optional[Dict] = None  # 参数优化结果（仅优化模式）
```

### 2.3 数据源模块

#### 2.3.1 统一接口 — data/fetcher.py

```python
"""
数据源工厂，根据 market 类型自动选择对应的数据源。
所有方法返回统一的数据模型。
"""
from abc import ABC, abstractmethod
from typing import List, Callable
from data.models import Symbol, Candle, Market, Interval
import config

class DataFetcher(ABC):
    """数据源抽象基类，所有数据源必须实现以下方法"""

    @abstractmethod
    async def get_symbols(self, query: str = "") -> List[Symbol]:
        """搜索/获取交易品种列表"""
        pass

    @abstractmethod
    async def get_klines(self, symbol: str, interval: Interval,
                         limit: int = 500) -> List[Candle]:
        """获取历史K线数据"""
        pass

    @abstractmethod
    async def subscribe_realtime(self, symbol: str, interval: Interval,
                                  callback) -> None:
        """订阅实时K线推送（WebSocket或轮询）"""
        pass

    @abstractmethod
    async def unsubscribe(self, symbol: str) -> None:
        """取消订阅"""
        pass

def get_fetcher(market: Market) -> DataFetcher:
    """工厂方法，根据市场类型返回对应的数据源实例"""
    if market == Market.CRYPTO:
        if config.CRYPTO_EXCHANGE == "okx":
            return OKXFetcher()
        return BinanceFetcher()
    elif market == Market.US or market == Market.HK:
        return YahooFetcher()
    elif market == Market.CN:
        return EastMoneyFetcher()
```

#### 2.3.2 OKX 数据源 — data/okx.py

**REST API 端点（公开数据，无需API Key）**：

| 接口 | 用途 | URL |
|------|------|-----|
| 获取交易品种 | 品种列表 | `GET /api/v5/public/instruments?instType=SPOT` |
| 历史K线 | K线数据 | `GET /api/v5/market/candles?instId=BTC-USDT&bar=1H&limit=100` |
| 更早历史K线 | 分页拉取 | `GET /api/v5/market/history-candles?instId=BTC-USDT&bar=1H&after={ts}&limit=100` |
| 最新行情 | 实时价格 | `GET /api/v5/market/ticker?instId=BTC-USDT` |
| 资金费率 | 合约费率 | `GET /api/v5/public/funding-rate?instId=BTC-USDT-SWAP` |
| 持仓量 | 合约OI | `GET /api/v5/public/open-interest?instId=BTC-USDT-SWAP` |
| 多空比 | 多空持仓 | `GET /api/v5/rubik/stat/contracts/long-short-account-ratio?ccy=BTC&period=1H` |

**WebSocket 订阅（实时K线）**：

```python
# 连接地址: wss://ws.okx.com:8443/ws/v5/public
# 订阅消息:
{
    "op": "subscribe",
    "args": [{"channel": "candle1H", "instId": "BTC-USDT"}]
}

# 推送数据格式:
{
    "arg": {"channel": "candle1H", "instId": "BTC-USDT"},
    "data": [
        ["1711324800000", "67234.5", "67890.0", "67100.0", "67650.25", "1234.56", "83234567.8", "83234567.8", "1"]
        # [ts(ms), open, high, low, close, vol(交易量), volCcy(计价货币成交量), volCcyQuote(USDT成交额), confirm(0未完结/1已完结)]
    ]
}
```

**OKX K线周期映射**：

| 平台周期 | OKX bar参数 | OKX WS channel |
|---------|------------|----------------|
| 1m | 1m | candle1m |
| 5m | 5m | candle5m |
| 15m | 15m | candle15m |
| 30m | 30m | candle30m |
| 1H | 1H | candle1H |
| 4H | 4H | candle4H |
| 1D | 1D | candle1D |
| 1W | 1W | candle1W |
| 1M | 1M | candle1M |

**实现要点**：
- WebSocket 需要每 25 秒发送一次 `"ping"` 保持连接
- REST API 限频：20次/2秒
- K线数据单次最多返回 **100 根**（不是1440），需要用 `after` 参数分页拉取更早的历史数据；更早的历史用 `/api/v5/market/history-candles` 接口
- 将 OKX 的数组格式转换为 `Candle` 对象

**注意**：

### 2.4 指标系统

#### 2.4.1 内置指标引擎 — indicators/builtin.py

用 NumPy 实现以下 20+ 指标，每个指标是一个纯函数，输入 OHLCV 数组，输出计算结果数组：

**趋势类（主图叠加）**：

| 指标 | 函数签名 | 默认参数 |
|------|---------|---------|
| MA | `calc_ma(close, period)` | period=20（单次调用计算一条线；前端默认同时添加 5,10,20,60 四条，每条独立调用） |

**后端实现方式**：
1. `parser.py` 用 Python AST 模块解析 OpenScript 代码
2. 将 OpenScript 转译为受限的 Python 代码
3. `executor.py` 在沙箱环境中执行转译后的代码
4. 沙箱白名单：只允许数学运算和内置函数，禁止 import、文件操作、网络访问
5. 执行超时限制：单次计算最多 5 秒
6. **重要**：OpenScript 中的 `open` 变量（开盘价）与 Python 内置 `open()` 函数同名。转译时必须确保内置 `open()` 被移除/屏蔽，在沙箱 globals 中只注入 OHLCV 数据数组，不暴露任何内置危险函数

**白名单库**：numpy, pandas, math, statistics
**禁止**：import os/sys/subprocess, open(), exec(), eval(), __import__
**执行超时**：10秒

# 用户Python代码示例
import numpy as np

def calculate(open, high, low, close, volume):
    """
    输入: OHLCV numpy数组
    返回: dict, key=指标名, value=数组或dict
    """
    # 自定义波动率指标
    returns = np.diff(np.log(close))
    vol_20 = np.array([np.std(returns[max(0,i-20):i]) * np.sqrt(252)
                        for i in range(1, len(returns)+1)])

    # 自定义信号
    signal = np.where(vol_20 > 0.5, 1, 0)  # 高波动标记

    return {
        "plots": [
            {"name": "年化波动率", "data": vol_20, "overlay": False, "color": "#FF6B6B"},
        ],
        "shapes": [
            {"name": "高波动", "data": signal, "style": "triangle_up", "color": "#FFD700"}
        ]
    }

**两条链路的关系：**
- 两条链路**完全独立**，可同时运行，互不阻塞
- 链路A的结果显示在右栏**上方表格区**
- 链路B的结果显示在右栏**下方卡片区**
- 两个区域之间用分割线 `── AI 智能推荐 ──` 隔开
- 如果只点了规则筛选没点AI分析，下方卡片区显示提示："点击 [🤖AI分析] 获取AI推荐"
- 如果只点了AI分析没配规则，上方表格区显示提示："配置左侧条件后点击 [🔍开始筛选]"

**评分权重公式说明：**

| 维度 | 权重 | 数据来源 | 说明 |
|------|------|---------|------|
| 新闻情绪 `sentiment_score` | 40% | LLM 返回的 -1~1 映射到 0~100 | 核心驱动因子 |
| 技术面 `technical_score` | 35% | 后端指标计算（金叉/突破/超卖等信号加分） | 量化验证 |
| 新闻热度 `news_volume_score` | 15% | 相关新闻数量归一化 | 关注度越高权重越高 |
| 影响程度 `impact_score` | 10% | LLM 标注的 高/中/低 映射到 100/60/30 | 事件重要性 |

**错误处理：**

| 异常场景 | 处理方式 |
|---------|---------|
| 新闻源全部超时 | 返回 `{"error": "news_fetch_failed", "message": "新闻数据获取失败，请稍后重试"}` |
| 单个新闻源失败 | 跳过该源，用其余源的数据继续分析（降级不中断） |
| LLM API 超时（30s） | 重试1次，仍失败返回 `{"error": "llm_timeout"}` |
| LLM 返回格式异常 | 解析失败则返回空推荐 + 日志报错 |
| LLM API Key 未配置 | 前端点击 [AI分析] 时弹窗提示 "请先在设置中配置 LLM API Key"，不发请求 |

### 4.4 WebSocket 客户端

```javascript
// frontend/js/websocket.js

class WSClient {
    constructor() {
        this.ws = null;
        this.handlers = {};         // type -> callback
        this.reconnectTimer = null;
        this.reconnectDelay = 3000; // 初始重连延迟 3s
        this.maxReconnectDelay = 30000; // 最大延迟 30s
    }

    connect() {
        this.ws = new WebSocket(`ws://${location.host}/ws`);
        this.ws.onopen = () => {
            this.reconnectDelay = 3000;  // 连接成功，重置延迟
            updateConnectionStatus("connected"); // 更新状态灯为绿色
        };
        this.ws.onmessage = (event) => {
            const msg = JSON.parse(event.data);
            const handler = this.handlers[msg.type];
            if (handler) handler(msg);
        };
        this.ws.onclose = () => {
            updateConnectionStatus("disconnected"); // 状态灯变红
            // 指数退避重连: 3s → 6s → 12s → 24s → 30s(上限)
            this.reconnectTimer = setTimeout(() => this.connect(), this.reconnectDelay);
            this.reconnectDelay = Math.min(this.reconnectDelay * 2, this.maxReconnectDelay);
        };
    }

    subscribe(symbol, interval) {
        this.send({ action: "subscribe", symbol, interval });
    }

    unsubscribe(symbol) {
        this.send({ action: "unsubscribe", symbol });
    }

    switch(symbol, interval) {
        // 自动取消旧订阅 + 创建新订阅（服务端原子操作，见 WS 协议 "switch" action）
        this.send({ action: "switch", symbol, interval });
    }

    on(type, callback) {
        this.handlers[type] = callback;
    }

    send(data) {
        if (this.ws?.readyState === WebSocket.OPEN) {
            this.ws.send(JSON.stringify(data));
        }
    }
}

// 使用示例
const ws = new WSClient();
ws.connect();

ws.on("kline", (msg) => {
    chart.updateData(msg.data);           // 更新K线
    updateIndicators(msg.indicators);      // 更新指标值
});

ws.on("alert", (msg) => {
    showAlertNotification(msg);            // 显示警报
    playAlertSound();                      // 播放声音
});

ws.on("backtest_progress", (msg) => {
    updateBacktestProgress(msg);           // 更新回测进度条
});

ws.on("backtest_complete", (msg) => {
    onBacktestComplete(msg);               // 回测完成，加载完整报告
});

ws.on("dashboard_update", (msg) => {
    updateDashboard(msg);                  // 更新仪表盘
});
```

**AI 分析 Loading 交互（重点）：**

由于 LLM 调用耗时较长（5~20秒），需要给用户明确的进度反馈：

```
阶段1（0~3秒）：  右栏AI区域显示
                  "🔄 正在采集最新新闻..."

阶段2（3~15秒）： 文字变为
                  "🤖 AI正在分析 47 条新闻..."
                  （47 = 实际采集到的新闻数量，通过中间响应更新）

阶段3（完成）：    隐藏loading，渲染推荐卡片
```

**实现方案：** 由于 REST API 不适合推送中间状态，采用以下方案之一（推荐方案A）：

- **方案A（推荐）**：`POST /api/screener/ai-analyze` 返回 `{"task_id": "ai_001"}`，前端轮询 `GET /api/screener/ai-status/{task_id}` 获取状态 `{status: "collecting_news" | "analyzing" | "scoring" | "done", progress: "采集到47条新闻", result: [...]}`
- **方案B**：通过 WebSocket 推送进度，`ws.on("screener_progress", (msg) => updateProgress(msg))`

**API Key 未配置的前置检查：**

```javascript
// screener.js — 点击[🤖AI分析]时的前置检查
async function onClickAIAnalyze() {

    // 根据当前 LLM 提供商检查对应的 API Key
    const keyField = settings.llm_provider === 'qwen' ? 'qwen_api_key' : 'deepseek_api_key';
    if (!settings[keyField] || settings[keyField] === '****') {
        showToast("请先在设置(⚙)中配置 LLM API Key", "warning");
        return;
    }
    // 通过检查，开始AI分析
    startAIAnalysis();
}
```

### 4.14 浏览器通知权限流程

```javascript
// 通知权限请求时机和流程
// 1. 首次创建警报时检查权限
// 2. 设置页面点击 [请求权限] 按钮

async function requestNotificationPermission() {
    if (!("Notification" in window)) {
        showToast("当前浏览器不支持桌面通知", "warning");
        return false;
    }
    if (Notification.permission === "granted") return true;
    if (Notification.permission === "denied") {
        showToast("通知权限已被拒绝，请在浏览器设置中手动开启", "error");
        return false;
    }
    // 弹出系统授权弹窗
    const result = await Notification.requestPermission();
    return result === "granted";
}

// 创建警报时自动检查
async function onCreateAlert(alertData) {
    if (alertData.notify_browser) {
        const granted = await requestNotificationPermission();
        if (!granted) {
            showToast("提示：浏览器通知未授权，警报将仅通过声音/Webhook通知", "info");
        }
    }
    // ... 继续创建警报
}
```

### 4.16 Loading / 空状态 / 错误状态 UI

| 场景 | UI 表现 |

| 数据源降级 | Toast 提示 "OKX 连接失败，已自动切换至 Binance 备用源" |

**Toast 通知组件规格：**
- 位置：页面右上角堆叠
- 类型：info（蓝）/ success（绿）/ warning（黄）/ error（红）
- 持续：3秒自动消失，error 类型需手动关闭
- 最多同时显示 3 条

## 五、数据库设计 (SQLite)

```sql
-- K线历史数据（按市场和周期分表，在 database.py 初始化时动态创建）
-- 表名示例: klines_crypto_1H, klines_us_1D, klines_cn_1D, klines_hk_1D 等
-- 需要创建的表: 4个市场 × 9个周期 = 最多36张表（按需创建，不必一次全建）
CREATE TABLE klines_{market}_{interval} (
    symbol TEXT NOT NULL,
    timestamp INTEGER NOT NULL,
    open REAL NOT NULL,
    high REAL NOT NULL,
    low REAL NOT NULL,
    close REAL NOT NULL,
    volume REAL NOT NULL,
    turnover REAL DEFAULT 0,
    PRIMARY KEY (symbol, timestamp)
);
-- 索引
CREATE INDEX idx_klines_symbol ON klines_{market}_{interval}(symbol, timestamp);

-- 自选列表
CREATE TABLE watchlist (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    symbol TEXT NOT NULL,
    market TEXT NOT NULL,
    name TEXT,
    sort_order INTEGER DEFAULT 0,
    added_at INTEGER NOT NULL,
    UNIQUE(symbol, market)           -- 同一市场内品种不重复，不同市场可有同代码
);

-- 警报配置
CREATE TABLE alerts (
    id TEXT PRIMARY KEY,
    symbol TEXT NOT NULL,
    market TEXT NOT NULL,
    condition_type TEXT NOT NULL,
    condition_json TEXT NOT NULL,     -- JSON 格式的条件详情
    message TEXT,
    notify_methods TEXT DEFAULT '["browser","sound"]',  -- JSON数组: "browser" | "sound" | "webhook"
    label TEXT DEFAULT '',           -- 备注标签
    repeat_mode TEXT DEFAULT 'once', -- once | every_time
    cooldown INTEGER DEFAULT 300,
    enabled INTEGER DEFAULT 1,
    created_at INTEGER NOT NULL,
    updated_at INTEGER
);

-- 警报触发历史
CREATE TABLE alert_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    alert_id TEXT NOT NULL,
    symbol TEXT NOT NULL,
    market TEXT NOT NULL,             -- 冗余存储市场，方便按市场筛选历史
    triggered_at INTEGER NOT NULL,
    price REAL,
    message TEXT
);

-- 回测报告
CREATE TABLE backtest_reports (
    id TEXT PRIMARY KEY,
    strategy_name TEXT NOT NULL,
    symbol TEXT NOT NULL,
    interval TEXT NOT NULL,
    start_date TEXT,
    end_date TEXT,
    config_json TEXT,                -- 回测配置
    result_json TEXT,                -- 完整回测结果（JSON）
    created_at INTEGER NOT NULL
);

-- 自定义公式/策略
CREATE TABLE formulas (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    description TEXT DEFAULT '',     -- 公式描述/备注
    code TEXT NOT NULL,
    mode TEXT DEFAULT 'openscript',  -- openscript | python
    type TEXT DEFAULT 'indicator',   -- indicator | strategy
    created_at INTEGER NOT NULL,
    updated_at INTEGER
);

-- 新闻缓存
CREATE TABLE news_cache (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    source TEXT NOT NULL,
    title TEXT NOT NULL,
    content TEXT,
    url TEXT,
    sentiment REAL,                  -- AI 情绪评分 (-1 到 1)
    symbols TEXT,                    -- 关联的品种（JSON数组）
    published_at INTEGER,
    analyzed_at INTEGER
);

-- AI选股异步任务
CREATE TABLE screener_tasks (
    task_id TEXT PRIMARY KEY,
    market TEXT NOT NULL,
    status TEXT DEFAULT 'collecting_news',  -- collecting_news | analyzing | scoring | done | error
    progress TEXT,                          -- 人类可读进度描述
    result_json TEXT,                       -- 完整结果（仅 done 时有值）
    error TEXT,                             -- 错误信息（仅 error 时有值）
    created_at INTEGER NOT NULL,
    completed_at INTEGER
);

-- 用户配置（KV 存储，运行时配置写入此处，优先级高于 config.py 默认值）
CREATE TABLE config (
    key TEXT PRIMARY KEY,
    value TEXT NOT NULL
);
```

"""
一键启动脚本

import uvicorn
import webbrowser
import threading
import time

def open_browser():
    time.sleep(2)

    threading.Thread(target=open_browser, daemon=True).start()
    uvicorn.run(
        "backend.main:app",

        port=8888,
        reload=True,
        log_level="info"
    )

## 九、关键注意事项

1. **OKX WebSocket 心跳**：每 25 秒发送 `"ping"`，否则服务器断开
2. **东方财富接口无认证**：直接 HTTP 请求即可，但需要设置合适的 User-Agent
3. **Yahoo Finance 限频**：避免短时间大量请求，建议用缓存
4. **公式沙箱安全**：严格限制 Python 执行环境，禁止文件/网络/系统调用
5. **数据库分表**：K线按市场和周期分表存储，避免单表过大
6. **前端性能**：KLineChart Pro 万级K线流畅，但指标过多时注意渲染性能
7. **错误处理**：所有数据源请求都要有超时和重试机制
8. **配置热更新**：修改 config.py 后尽量不需要重启服务
9. **LLM API 调用**：做好 Token 用量统计和异常处理，避免意外高额账单
10. **CORS 配置**：开发环境允许 localhost，生产环境按需限制
11. **交易时段管理**：各市场交易时间不同，轮询调度需根据市场判断：
    - 加密货币：7×24 全天候，持续轮询
    - A股：工作日 9:15-11:30, 13:00-15:00（北京时间）
    - 港股：工作日 9:30-12:00, 13:00-16:00（北京时间）
    - 美股：工作日 21:30-04:00 夏令时 / 22:30-05:00 冬令时（北京时间）
    - 非交易时段停止行情轮询以节省资源，但保留最后一次数据在缓存中供前端展示
12. **Symbol 命名统一**：全平台统一使用 OKX 格式（带横杠），如 `BTC-USDT`。Binance 备用数据源内部做 `BTC-USDT` ↔ `BTCUSDT` 的转换
13. **前端库引入**：KLineChart 用 UMD 版本（`klinecharts.init()` 而非 ES Module import），CodeMirror 6 用 ESM CDN，Chart.js 用 CDN
14. **LLM 统一调用**：DeepSeek 和通义千问都用 OpenAI 兼容模式（`openai` Python库），只需切换 `base_url` 和 `api_key`，不需要写两套调用逻辑

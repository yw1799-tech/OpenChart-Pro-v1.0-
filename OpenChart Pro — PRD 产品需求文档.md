## 1. 产品概述

### 1.1 产品定位

构建一个 **AI 辅助交易决策平台**，命名为 **OpenChart Pro**。平台围绕"**看得清 → 选得准 → 盯得住 → 做决策**"四个环节，提供从专业图表分析到 AI 驱动候选推荐、策略信号监控、持仓建议的完整决策链路。

支持加密货币、美股、港股、A 股四大市场。后期可升级为自动交易下单。

### 1.2 两种市场模式

加密货币和股票的使用方式**根本不同**，平台区分两种运行模式：

**加密货币 — 固定币种 + 影响分析模式**

```
固定 6 币种（BTC / ETH / SOL / DOGE / BNB / XRP）
  │
  │  不需要"选什么币"，直接进入监控
  │
  ├─ 币圈新闻 → 这条新闻对我的 6 个币有什么影响？
  ├─ 链上数据 → 巨鲸买入/交易所流出哪个币？
  ├─ 宏观数据 → CPI/FOMC/非农公布后，对加密市场什么影响？
  ├─ 技术指标 → 6 个币的实时指标监控
  │
  └─→ 建议：加仓 BTC / 减仓 SOL / 观望 ETH...
```

**股票 — 筛选 + 候选池 + 监控模式**

```
全市场几千只股票
  │
  ├─ 新闻事件 / 异动扫描 / 技术形态 / 宏观主题 → 筛选推荐
  ├─ 手动添加
  │
  ▼
候选池（待监控标的）
  │
  ├─ 绑定策略深度监控
  ├─ 宏观数据 → CPI/非农/PMI 公布后对哪些股票/板块有影响？
  │
  ▼
触发信号 → 开仓提醒
  │
  ▼
持仓管理 → 操作建议
  │
  ▼
未来：自动下单（预留）
```

| 功能 | 加密货币（固定 6 币种） | 股票（美股/港股/A 股） |
|------|----------------------|---------------------|

| 仪表盘 | 专属（恐惧贪婪/资金费率/链上数据） | 不使用 |

### 1.8 首次使用引导

| 步骤 | 用户看到什么 | 系统做什么 |
|------|-------------|----------|
| 首次打开 | 加密市场 BTC-USDT K 线（默认） | 自动加载 6 币种到自选列表，开始新闻采集 |
| 切换到股票 | 空的自选列表 + 提示"搜索品种添加到自选" | 引导用户搜索添加 |
| 24h 后 | 候选池出现 AI 推荐的股票 | 新闻采集 + 规则引擎已运行，符合条件的股票被推入 |
| 点击候选品种 | 提示"绑定策略开始监控" | 引导绑定策略 |
| 信号触发 | Toast 弹窗 + 声音 + "点击查看详情" | 跳转到信号标签页 |

> 关键：**不需要设置向导**，系统默认配置即可运行。加密市场开箱即用（6 币种自动监控），股票市场需要 24h 让新闻系统积累数据后才有推荐。

- 高级参数优化（Walk-Forward、蒙特卡洛）—— 同上
- 多租户 SaaS
- 原生移动端 App

## 2. 功能需求

### 2.1 F1 — K 线图表（Phase 1）

**功能描述**：以专业蜡烛图展示实时和历史 K 线数据，支持多种图表类型和时间周期。

| # | 需求 | 说明 |
|---|------|------|

| F1.2 | 历史 K 线 | 支持加载至少 500 根历史 K 线 |
| F1.3 | 时间周期 | 9 档：1m / 5m / 15m / 30m / 1H / 4H / 1D / 1W / 1M |
| F1.4 | 图表类型 | 实心蜡烛、空心蜡烛、美国线、面积图 |
| F1.5 | 涨跌配色 | 国际标准 / 中国标准，可切换 |
| F1.6 | 画线工具 | 趋势线、斐波那契、矩形、通道、文字标注 |
| F1.7 | 画线持久化 | 按品种保存，切换品种时加载 |
| F1.8 | 周期限制 | 加密货币支持全部 9 档周期；A 股隐藏 1m 和 4H；美股/港股隐藏 1m |
| F1.9 | 历史自动加载 | 图表往左拖动到边缘时，自动加载更早的历史 K 线（无限滚动） |

**验收标准**：浏览器能看到 BTC-USDT 实时 K 线，能切换周期，键盘 1~9 快速切换，往左拖动自动加载更多历史数据。

### 2.5 F5 — 新闻分析与宏观影响引擎（Phase 3）

**功能描述**：对采集到的新闻和宏观数据进行**分级处理**，绝大部分由免费的规则引擎完成，LLM 只在最关键时刻介入。同时分析每条重要新闻/数据**对持仓和关注品种的具体影响**。

#### 5A. 新闻分级处理流程

```
5000条/天进入
  │
  ▼
L1 去重（URL + SimHash）       → 丢弃 60%，剩 ~2000 条          免费
  │
  ▼
L2 规则引擎（关键词 + 来源）   → 打分 + 标签 + 品种关联          免费
  │                               ★★ 以下到此结束，直接入库
  │                               ★★★ 以上约 600 条继续
  ▼
L3 影响分析 + 候选推荐         → 分析对持仓/关注币种的影响       免费
  │                               股票：高分品种推荐入候选池
  │                               仅 ★★★★+ 约 50 条继续
  ▼
L4 LLM 深度解读                → 详细分析（理由/风险/价位）      ~$0.001/次
  │
  总计：约 $1.5/天 ≈ $45/月
```

**核心原则**：
- 推荐入池和影响判定**不依赖 LLM**，规则引擎就够了
- LLM 仅用于"深度解读"（用户点击或极高价值事件）
- 加密货币新闻重点分析**对固定 6 币种的影响**，不做选币
- 股票新闻才做候选推荐

#### 5B. 宏观数据影响分析

**关键能力**：宏观数据公布后，自动判定对持仓品种和关注品种的影响方向与程度。

| 宏观数据 | 对加密 6 币种的影响规则 | 对股票的影响规则 |
|---------|----------------------|----------------|
| **CPI 高于预期** | 全体利空（加息预期）→ 6 币种提示"注意风险" | 成长股/科技股利空，银行股可能利好 |
| **CPI 低于预期** | 全体利好（降息预期）→ 6 币种提示"利好" | 成长股利好 |
| **FOMC 加息** | 全体利空 → 建议减仓或观望 | 全体利空，地产/高负债股更敏感 |
| **FOMC 降息** | 全体利好 → 可考虑加仓 | 全体利好，成长股更受益 |
| **非农强劲** | 利空（经济强→不急降息） | 周期股利好，成长股利空 |
| **非农疲软** | 利好（经济弱→可能降息） | 防御股利好，周期股利空 |
| **GDP 超预期** | 弱关联，偏利好 | 周期股利好 |
| **PMI 制造业** | 弱关联 | 制造业/工业股直接影响 |
| **零售数据** | 弱关联 | 消费/零售板块直接影响 |
| **芯片出口数据** | 无关 | 半导体股（NVDA/AMD）直接影响 |
| **房屋数据** | 无关 | 地产/建材板块直接影响 |
| **贸易政策/关税** | 弱关联 | 出口型企业直接影响 |

上述规则由**规则引擎 + 影响映射表**驱动，不需要 LLM。数据公布后自动：
1. 对比实际值 vs 预期值 → 判定超预期/符合/不及预期
2. 查影响映射表 → 判定对持仓/关注品种的影响方向
3. 推送影响摘要到前端（如"CPI 超预期，加密市场承压，建议关注减仓"）
4. 仅重大偏离时调用 LLM 做深度解读

| # | 需求 | 说明 |
|---|------|------|
| F5.1 | 规则引擎为主力 | 关键词权重 + 来源可信度评分，覆盖 70%+ 场景，完全免费 |
| F5.2 | 重要性评分 | 1~5 星（基于关键词权重 × 来源可信度 + 持仓加分） |
| F5.3 | 品种关联 | 规则引擎实体匹配识别涉及品种（不需要 LLM） |
| F5.4 | 情绪判定 | 关键词模式判定 bullish / bearish / neutral |
| F5.5 | **加密影响分析** | 新闻/宏观数据对固定 6 币种的影响。输出格式：`{币种, 方向(利好/利空/中性), 强度(1-5), 理由}` |
| F5.6 | **股票影响分析** | 宏观数据对持仓股票和候选池股票的影响（规则 + 板块映射） |
| F5.7 | **宏观数据速报** | 数据公布后秒级推送：实际值 vs 预期值 + 影响判定 + 操作建议 |
| F5.8 | 股票候选推荐 | ★★★+ 新闻涉及的**股票**自动推入候选池（加密不需要） |
| F5.9 | LLM 深度解读 | 仅 ★★★★+ 或用户点击时调用 |
| F5.10 | 持仓快速通道 | 涉及持仓品种的新闻自动 +3 分 |
| F5.11 | 成本控制 | 跟踪每次 LLM 调用成本并累计。**日预算上限 $5（可在设置中调整）**；超 80% 时告警；超 100% 时当日停止 LLM 调用（规则引擎照常运行，不丢新闻）。用户可在设置中手动提高上限或关闭限制 |
| F5.12 | 前端展示 | 快讯流 + 影响标签 + 可点击"AI 解读" |

| 权重 | 关键词示例 |
|------|----------|

|------|--------|------|

| 社交媒体（微博/推特） | 0.5 | 0 |
| 未知来源 | 0.3 | -2 |

**v1 只监控 3 项核心宏观数据**（CPI / FOMC / 非农），其他后续版本扩展。

**偏差判定标准**：

```
偏差 = (实际值 - 预期值) / 预期值

符合预期:  |偏差| < 0.5%  →  中性，不触发信号
轻微偏离:  0.5% ≤ |偏差| < 1.0%  →  轻微影响，仅通知不触发信号
明显偏离:  |偏差| ≥ 1.0%  →  触发影响分析 + 可能触发信号

FOMC 利率决议特殊处理：
  加息/降息 vs 市场预期一致 → 中性
  意外加息/降息（与预期不符）→ 强影响
  措辞偏鹰/偏鸽（需 LLM 判定）→ 轻微影响
```

**影响映射表（明显偏离时生效）**：

| 数据 | 情况 | 加密 6 币种 | 科技/成长股 | 银行/金融股 | 消费股 | 影响强度 |
|------|------|-----------|-----------|-----------|--------|---------|
| CPI | 高于预期 | 利空 ↓ | 利空 ↓ | 利好 ↑ | 利空 ↓ | 偏差越大越强 |
| CPI | 低于预期 | 利好 ↑ | 利好 ↑ | 利空 ↓ | 利好 ↑ | 偏差越大越强 |
| FOMC | 意外加息 | 强利空 ↓↓ | 强利空 ↓↓ | 利好 ↑ | 利空 ↓ | 强 |
| FOMC | 意外降息 | 强利好 ↑↑ | 强利好 ↑↑ | 利空 ↓ | 利好 ↑ | 强 |
| 非农 | 强劲（高于预期） | 利空 ↓ | 中性 | 利好 ↑ | 利好 ↑ | 中 |
| 非农 | 疲软（低于预期） | 利好 ↑ | 中性 | 利空 ↓ | 利空 ↓ | 中 |
| *任何* | **符合预期** | **中性** | **中性** | **中性** | **中性** | **无** |

> 影响窗口：数据公布后 24 小时内有效，之后不再影响评分和信号。

**候选池评分公式（0-100）**：

```
score = 新闻热度 × 0.4 + 价格动量 × 0.3 + 技术信号 × 0.2 + 异动排行 × 0.1

各分量计算方式：

新闻热度（0-100）：
  = min(100, 24h内★★★+新闻数量 × 20 + 最高星级 × 10)
  × 时间衰减系数（最近1h=1.0, 6h=0.7, 12h=0.5, 24h=0.3）

价格动量（0-100）：
  = min(100, abs(24h涨跌幅%) × 10 + (成交量/20日均量 - 1) × 20)
  注：涨和跌都算动量，方向由情绪判定

技术信号（0-100）：
  = 0（无信号） / 50（有信号但置信度低） / 100（有高置信度信号）

异动排行（0-100）：
  = 100 如果出现在数据源涨幅/成交量 Top 10
  = 50 如果出现在 Top 20
  = 0 如果未上榜
```

| 绑定方式 | 操作 | 示例 |

**信号置信度计算（各策略独立定义基础分 + 加减分项）**：

```
通用加减分项（所有策略共享）：
  + 10  如果 RSI 在合理区间（30-70，非超买超卖）
  + 15  如果价格在 200 日均线上方（上升趋势中）
  + 10  如果成交量 > 20 日均量 1.5 倍（有量确认）
  - 10  如果有近期利空新闻（★★★+ bearish）
  - 15  如果宏观数据刚公布且对该品种利空

示例：
  均线金叉策略：基础分 55 + 通用加减分 → 如果 RSI 正常(+10) + 趋势向上(+15) = 80 → 触发
  RSI 背离策略：基础分 50 + 通用加减分 → 如果无量配合(-0) + 利空新闻(-10) = 40 → 不触发

最终置信度 = clamp(0, 100, 基础分 + ∑加减分)
```

**布局原则**：图表是核心，其他面板辅助。1366px 笔记本也能正常使用。

```
┌─────────────────────────────────────────────────────────────────────┐
│  工具栏 (44px)                                                       │
│  [市场▼] [搜索...] [1m|5m|15m|30m|1H|4H|1D|1W|1M] [指标] [画线]    │
│  [⚙设置]                                                  🟢实时   │
├──────────┬──────────────────────────────────────────────────────────┤
│ 侧边栏   │                                                          │
│ (160px)  │              主图表区域                                    │
│ 可折叠   │              蜡烛图 + 叠加指标                             │
│          │                                                          │
│ ┌自选列表┐│──────────────────────────────────────────────────────────│
│ │BTC    ││              副图区域                                     │
│ │ETH    ││              MACD / RSI / KDJ                            │
│ │NVDA   ││                                                          │
│ │[+添加]││                                                          │
│ └────────┘│                                                          │
├──────────┴──────────────────────────────────────────────────────────┤
│  底部面板 (可收起, 默认 250px, 一次只显示一个标签内容)                  │
│  [新闻快讯] [候选池] [信号] [持仓] [回测] [仪表盘]                   │
└─────────────────────────────────────────────────────────────────────┘
```

**与 v2 布局的区别**：
- **去掉右侧信息面板**（品种信息改为鼠标悬停图表时 Tooltip 显示，或点击品种弹出详情浮窗）
- 侧边栏收窄到 160px，**可折叠**（折叠后只显示图标，图表占满宽度）
- 底部面板缩小到 250px，**一次只显示一个标签页内容**
- 活跃信号和持仓建议在底部标签页中查看，不额外占用空间
- 信号触发时通过 Toast + 声音通知，用户点击 Toast 跳转到信号标签页

### 3.2 底部面板标签页

| 标签页 | 内容 |
|--------|------|
| **新闻快讯** | 实时滚动的新闻流，标注星级/情绪/关联品种，高价值新闻高亮，可点击展开 AI 深度解读 |
| **候选池** | 左栏：候选品种列表（评分/来源/状态），右栏：选中品种的监控详情（绑定策略/最新信号/新闻） |
| **信号历史** | 所有策略信号的时间线，含触发条件、置信度、建议操作 |
| **持仓管理** | 持仓列表 + 浮动盈亏 + 最新建议 + 建议历史 |
| **回测报告** | 左栏策略选择 + 右栏报告（资金曲线/统计/交易记录） |
| **仪表盘** | 恐惧贪婪、资金费率、持仓量、经济日历、链上数据 |

### 3.9 Toast 通知

| 属性 | 规格 |
|------|------|

## 4. 非功能需求

### 4.1 性能要求

| 场景 | 目标 |
|------|------|
| 500 根 K 线加载 | < 3 秒 |
| 新闻采集入库 | 发布后 < 30 秒入库（RSS 源受轮询间隔限制） |
| 规则引擎处理 | 单条 < 50ms，1000 条/小时无积压 |
| 新闻到前端展示 | 入库后 < 2 秒推送到前端 |
| LLM 深度分析 | < 15 秒（异步，不阻塞规则引擎和前端推送） |
| 宏观数据影响分析 | 数据公布后 < 60 秒完成影响判定并推送 |
| 5000 只股票指标计算 | < 1 秒 |
| 候选池重评分（全池） | < 30 秒 |
| 图表万级 K 线渲染 | 流畅 |

### 4.2 安全要求

| 领域 | 要求 |
|------|------|
| API Key 存储 | SQLite 存储，API 返回脱敏 |
| CORS | 开发 `*`，生产限制 |
| LLM 调用 | 成本监控 + 异常处理 |
| 新闻采集 | 合理频率，避免被源站封禁 |

### 4.3 容错要求

| 场景 | 超时 | 重试 |
|------|------|------|
| REST API | 10s | 3 次（指数退避） |
| WebSocket 断线 | - | 无限（上限 30s） |
| LLM API | 30s | 2 次 |
| 数据源故障 | - | 自动切备源 |
| 新闻源故障 | - | 单源故障不中断，用剩余源 |
| 候选池评分失败 | - | 保留上次评分，标记为"待更新" |

### 4.4 可用性

| 场景 | 要求 |
|------|------|
| 一键启动 | `python run.py`，2 秒后自动打开浏览器 |
| 新闻采集 | 后台自动启动，无需用户干预 |
| 候选池维护 | 自动推入/淘汰，用户可手动干预 |
| 配置热更新 | 修改设置无需重启 |

## 5. 产品路线图

### Phase 1 — 核心基建

**交付物**：K 线图表 + 数据源 + WebSocket + 搜索 + 设置  
**验收标准**：浏览器能看到 BTC-USDT 实时 K 线，能切换周期、搜索品种、保存设置。

### Phase 2 — 指标和多市场

**交付物**：20+ 指标 + 美股/港股/A 股数据 + 自选列表  
**验收标准**：能看四大市场 K 线，能叠加多个指标。

### Phase 3A — 基础新闻 + 候选池框架（最小可用）

**交付物**：10 源新闻采集 + URL 去重 + 规则引擎关键词匹配 + 新闻面板 + 候选池手动添加  
**验收标准**：
- 新闻面板有数据滚动更新（10 个源）
- 新闻标注星级和关联品种
- 用户可手动添加股票到候选池
- 异动排行榜数据拉取并显示

> **里程碑意义**：Phase 1 + 2 + 3A 完成后，系统已可作为"带新闻的多市场看盘工具"使用。

### Phase 4 — 策略信号 + 加密监控

> **依赖**：Phase 2（技术指标）。加密监控不依赖 Phase 3A。

**交付物**：内置策略 + 加密 6 币种自动监控 + 候选池品种绑定策略 + 信号提醒  
**验收标准**：
- 加密 6 币种系统启动后自动开始监控，触发信号时通知
- 候选池股票可绑定策略（一对多、多对一灵活绑定）
- 触发信号时多渠道通知
- 信号包含完整建议（方向/仓位/止损/止盈/理由 + 置信度）

> **里程碑意义**：Phase 4 完成后，加密用户已可获得完整的交易信号服务。

### Phase 3B — 新闻深度 + 候选池自动化 + 宏观影响

**交付物**：SimHash 去重 + LLM 深度解读 + 候选池自动推入/淘汰/重评分 + 宏观数据影响分析  
**验收标准**：
- 高价值新闻自动推荐股票入候选池
- 候选池每小时自动重评分 + 自动淘汰
- CPI/FOMC/非农公布后 60 秒内推送影响分析
- 宏观数据影响覆盖加密 6 币种 + 持仓股票

> **为什么 3B 在 4 之后**：Phase 4 的加密信号是最高优先级功能，不应被新闻深度分析阻塞。Phase 3B 的候选池自动化和宏观影响让股票端体验完整。

### Phase 5 — 持仓建议 + 回测

**交付物**：持仓管理 + 操作建议 + 基础回测  
**验收标准**：
- 录入持仓后持续收到操作建议
- 新闻利空时自动推送减仓/平仓建议
- 能对内置策略做基础回测

### Phase 6 — 加密仪表盘

**交付物**：链上数据 + 情绪指标 + 经济日历  
**验收标准**：仪表盘各项数据正常展示。

### Phase 7 — 自动交易（预留）

**交付物**：BrokerAdapter 接口 + 模拟模式 + 风控  
**验收标准**：待定。

### 交付顺序总览

```
Phase 1（核心基建）→ Phase 2（指标+多市场）→ Phase 3A（基础新闻+候选池框架）
                                                          │
                                              ┌───────────┴───────────┐
                                              ▼                       ▼
                                     Phase 4（策略信号+加密监控）  Phase 3B（新闻深度+宏观影响）
                                              │                       │
                                              └───────────┬───────────┘
                                                          ▼
                                              Phase 5（持仓建议+回测）
                                                          ▼
                                              Phase 6（加密仪表盘）
                                                          ▼
                                              Phase 7（自动交易预留）
```

> Phase 4 和 Phase 3B **可并行开发**，Phase 4 优先交付。

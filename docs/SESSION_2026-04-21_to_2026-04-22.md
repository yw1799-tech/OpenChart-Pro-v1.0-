## 问题 2：K 线往左拖不自动加载历史

### 症状
用户把 K 线缩小 + 向右拖动翻历史，数据不自动加载更多。反复刷新也无效。

### 诊断历程（多个 Bug 叠加）
1. **v1 错判**：我误以为 BOOT_ID 缓存问题，只调了资源版本号。失败。
2. **v2 错判**：以为 `ActionType.OnVisibleRangeChange` 和 `OnScroll` 是 KLineChart v10 API。查了 min.js 确认 v9.6.0 也有这些 ActionType。排除。
3. **v3 真正根因（Playwright 实测）**：
   - KLineChart 源码（压缩后 @9844）`loadMore` 回调触发条件是 `l===0 && this._more && !this._loading`
   - v9 `applyMoreData(data, more, callback)` 第二参数 `more` 是"是否还有更多"的门控
   - 前端代码传 `false` → `_more` 永久关闭 → loadMore 回调再也不触发
   - 空结果路径直接 `return` → `_loading` 卡在 true → 全链路锁死
4. **v4 发现**：`[loadMore]` 日志被 `app.js` 的 `console.log = noop` 吞掉了，看起来像没反应
5. **v5 触发阈值过严**：`from > 30` 才不触发 —— 用户拖动一次 from 减少约 75，6 次后 from=73 还没触发；加载完后视图又跳回 from=648，还需拖 6 次

### 最终修复
- 文件：[frontend/js/chart.js](../frontend/js/chart.js)
- `applyMoreData(filtered, true)` 保持 `_more=true`
- 空结果 `applyMoreData([], false)` 主动解锁 `_loading`
- 关键日志改 `console.warn`，不被 app.js 静默吞
- 触发阈值改为动态 `max(200, total * 0.25)` —— 前 25% 区域就预加载
- 暴露 `window._fetchMore('manual')` 供调试

### 顺带改进
- [backend/main.py:2550-2583](../backend/main.py#L2550-L2583) BOOT_ID 改成按 `frontend/js + frontend/css` 最大 mtime 计算 —— 改前端文件版本号立即变，浏览器必然重载，不用重启后端

### Playwright 实测验证
持续拖动 20 次，数据 700 → 1200 → 1700 → 2200，每 5 次拖动自动加载一批 500 根。

## 问题 3：策略信号确认后为什么不开仓

### 症状
大量 `ai_verdict=confirm` 信号（截图 8 条 BTC-USDT buy）无对应持仓/操作记录。

### 根因
1. BTC-USDT 已有持仓 → 走加仓分支（`_try_add_position`）
2. 加仓硬规则："持仓浮盈 ≥ 5%" 才加
3. BTC 当前浮盈 -0.02% ~ +0.98%（长期未达 5%）
4. **5 处 early return 完全静默**，在 `auto_trade_log` 不留任何记录 → 用户看不到"为什么没开"

### 修复
- 文件：[backend/trading/auto_trader.py](../backend/trading/auto_trader.py)
- 把 `_try_add_position` 的 4 处静默 return 改为 `_log_rejected`（浮盈不足 / 单仓上限 / 订单金额过小 / 现金不足）
- 把 `_handle_signal` 的 3 处静默 return 改为 `_log_rejected`（反向持仓 / 同股冷却 / 单日超限）

### 结果
以后被拒都会在"自动交易日志"里显示具体原因，例如：
> 加仓需持仓浮盈≥5%，当前 PnL=0.14%（long 均价75585.9000 vs 现价75694.7000）

## 问题 6：账户总盈亏显示错误

### 症状
前端显示账户盈亏 +$4035.86 (+4.04%)，但 BTC 浮盈只有 $39.86。

### 根因
旧算法 `pnl = 总资产 − 初始资金 = (cash + positions_value) − initial`。当持仓是手动录入时（走 portfolio_manager.add_position，不走 auto_trader），不扣 `cash_usd`，持仓市值凭空多出来，被误算为盈利。

### 修复
- 文件：[backend/main.py:2331-2461](../backend/main.py#L2331-L2461)
- 新算法：`pnl = 浮盈 + 已实现盈亏`
- 浮盈 = Σ 当前持仓 (市值 − 成本)
- 已实现 = 按 position_id 分组，**只统计有 close action 的闭环单** (out − in)
- 孤儿支出（有 open 无 close 且 position 已消失）单独标记 `orphan_amount_usd`，不计入 pnl
- 前端顶部摘要拆显示：`浮盈 +$X · 已实现 +$Y` + `⚠️ 孤儿交易 $Z` 提示

## 问题 8：信号 60 分钟后 stale 过期

### 症状（截图）
6 条"已过期"信号，理由"信号超过 60 分钟未验证，市场已变，跳过"。

### 根因
文件：[backend/main.py:571-653](../backend/main.py#L571-L653) `_verify_backfill_loop`
- 旧配置：批次间隔 180s、每批 10 条、每条 `sleep(3s)` → 峰值 3.3 条/分钟
- 半夜多品种+多周期同时出信号 → 堆积速度远高于验证速度
- 60 分钟一刀切 stale → 1H/4H/1D 这些长周期信号"市场没变"也被误杀

### 修复
| 参数 | 旧 | 新 |
|---|---|---|
| 批次间隔 | 180s | 60s |
| 每批上限 | 10 | 30 |
| 每条延迟 | 3s | 0.5s |
| stale 窗口 | 60min 一刀切 | 按周期动态：1m→30min · 5m→1h · **15m→2h** · 30m→3h · **1H→4h** · **4H→12h** · **1D→24h** · 1W→3d |

理论吞吐 60 × 60 / 0.5 = **3600 条/小时**，不再堆积。

# 开发 SOP

## 新功能开发流程
1. 先用一句话写清楚：这个功能解决谁的什么问题
2. 找到项目里最像的已有功能，参考它的结构
3. 列出要改的文件清单，贴给 AI 确认方案
4. 先写类型/接口/数据模型
5. 再写实现
6. 手动测三个场景：正常、边界、异常
7. 提交前对照 docs/checklist.md

## Bug 修复流程
1. 先复现（复现不了不改）
2. 用一句话描述：期望 vs 实际
3. 打开 docs/ai/debug-log.md，填当前 debug 状态
4. 定位根因（二分法 / 日志 / 断点）
5. 根因确认后再动手
6. 修完问自己：同类 bug 还会在哪出现
7. 把教训写进 docs/ai/lessons.md
8. 清空 debug-log.md

## 技术指标/算法类开发流程
1. 写注释说明公式/算法来源
2. 构造玩具数据，手算正确结果
3. 实现，跑玩具测试
4. 和权威实现对拍（如果有）
5. 可视化肉眼检查
6. 加不变量断言
7. 异常数据测试（全同、边界、空值、超长）

## 缠论相关开发流程
1. 只改 `chanlun_service.py`，不动 chan.py 引擎源码
2. 改参数前用 `chanpy_audit.py` 跑对比，确认 chan.py 和我们输出一致
3. 改完用 `full_audit.py` 跑全周期验证（1m~1W）
4. 检查买卖点有买有卖、数量合理
5. 在图表上肉眼检查笔/线段/买卖点是否贴合K线

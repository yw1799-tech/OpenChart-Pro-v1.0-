# 使用说明

## 这是什么

两份文档,帮你把一个"野路子项目"规范化,全程由 Cursor/VS Code 里的 AI 代你执行,你只要负责确认和做决定。

## 文件清单

1. **项目规范化执行指令.md** — 给 AI 读的执行脚本,12 个步骤
2. **规范化进度表.md** — 给你用的打勾清单

## 怎么用(三步)

### 第一步:把两份文档放进你的项目

打开 VS Code / Cursor,打开你的项目文件夹,把 `项目规范化执行指令.md` 和 `规范化进度表.md` 都拖进项目根目录。

### 第二步:打开 AI 对话框,复制下面这段话发给它

```
请按照项目根目录下的 `项目规范化执行指令.md` 里的步骤,从第 0 步开始,
一步一步带我做。每完成一步等我说"继续"再做下一步。
遇到需要我决定的事先问我,不要自作主张。
我是非技术人员,请用中文解释,命令告诉我粘贴到哪里。
每完成一步主动提醒我去 `规范化进度表.md` 打勾。
现在开始第 0 步。
```

### 第三步:跟着走

AI 会一步一步带你做。你要做的只有三件事:

1. **看它说什么**,不懂就追问"这是什么意思"
2. **确认后说"继续"**,AI 才会执行下一步
3. **每完成一步**,打开进度表打勾

## 重要提醒

- **第 0 步(git 快照)不能跳过**,这是你的安全网
- **一天做完不现实也没必要**,建议分 3 周:第一周做第 0-5 步,第二周做第 6-8 步,第三周做第 9-12 步
- **任何一步出问题**,告诉 AI "跑不起来了,回滚",它会帮你回到上一步
- **觉得某一步暂时不需要**,说"跳过"就行,以后再做
- **最关键的是第 0-5 步**,做完这 5 步项目就已经比 90% 的个人项目规范了

## 如果 AI 卡住或跑偏了

告诉它:"请重新读 `项目规范化执行指令.md`,我们现在应该在第 X 步,继续。"

## 如果你想改流程

这两份文档是你的,随便改。觉得某一步不合理就删,想加步骤就加。AI 会按你改过的版本执行。

---

祝规范化顺利。做完你会发现,规范化真正的价值不是项目变整洁了,而是**你开始用"工程师的思维"管理自己的项目**。这个思维一旦建立,以后每个新项目都会自动带上。

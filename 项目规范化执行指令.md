# 项目规范化执行指令

> **使用方法**：在 Cursor 或 VS Code 里打开你的项目，把这份文档也放进项目根目录，然后在 AI 对话框里说：
>
> **"请按照 `项目规范化执行指令.md` 里的步骤,从第 0 步开始,一步一步带我做。每完成一步等我确认 '继续' 再做下一步。遇到需要我决定的事先问我。我是非技术人员,请用中文解释,命令告诉我粘贴到哪里。"**

---

## 给 AI 的总体要求（AI 必读）

1. **用户是非技术人员**,所有解释用中文,不要假设用户懂代码
2. **每一步做完必须等用户说"继续"**,不要一次做完所有步骤
3. **每次改动前先说明**:要改什么、为什么、会不会有风险
4. **每一步都要让用户验证能跑**,不能跑就停下来修
5. **不要自作主张装新依赖或删文件**,任何写入/删除操作先问
6. **发现敏感信息**(密码、密钥、token)立即停止并告警
7. **每完成一步**,主动提醒用户在 `规范化进度表.md` 打勾
8. **用户说"跳过"**就跳过当前步骤,不要劝说

---

## 第 0 步:安全快照(必须第一步,不能跳过)

**目标**:在动任何东西之前,做一个 git 快照,让整个过程可以随时回滚。

**AI 要做的事**:

1. 先检查项目是否已经是 git 仓库:
   - 如果是,检查 `git status`,告诉我当前有没有未提交的改动
   - 如果不是,告诉我这个项目还没用 git,问我是否要初始化

2. 如果已经是 git 仓库且有未提交改动:
   - 让我确认这些改动是否要先提交
   - 然后执行 `git add . && git commit -m "chore: snapshot before regularization"`

3. 如果不是 git 仓库:
   - 执行 `git init`
   - 执行 `git add .`
   - 执行 `git commit -m "chore: initial snapshot before regularization"`

4. 完成后告诉我:快照已建立,以后任何一步出问题都可以用 `git reset --hard HEAD` 回到这里。

**验证**:让我运行 `git log --oneline -1`,应该能看到刚才的 commit。

**完成标志**:存在一个名为 "snapshot before regularization" 的 git 提交。

## 第 1 步:项目体检

**目标**:搞清楚项目现状,生成一份诊断报告。

**AI 要做的事**:

扫描项目根目录和所有主要文件夹,回答以下问题,把答案写到 `docs/audit.md`(如果 `docs/` 不存在就先建):

1. **项目类型**:这是一个什么类型的项目?(网页、脚本、API、桌面应用、数据分析...)
2. **技术栈**:主要用了哪些编程语言、框架、库?(从依赖文件如 package.json、requirements.txt、pyproject.toml、Gemfile 等判断)
3. **项目结构**:列出主要的文件夹和它们的作用(最多列 2 层深度)
4. **入口文件**:从哪里开始运行?(main.py、index.js、app.py 等)
5. **现有文档**:有没有 README?有没有其他文档?质量如何?
6. **依赖管理**:有没有规范的依赖管理文件?有没有锁文件(package-lock.json、poetry.lock 等)?
7. **版本控制**:有没有 .gitignore?是否合理?
8. **代码风格**:代码风格是否统一?有没有格式化工具配置(.prettierrc、.eslintrc、pyproject.toml 里的 black 配置等)?
9. **测试**:有没有测试文件?测试覆盖哪些部分?
10. **明显问题**:从文件结构和命名角度看,最明显的 3 个问题是什么?
11. **敏感信息风险**:有没有发现 .env 文件、API key、密码等敏感信息?如果有,它们是否被 git 追踪了?
12. **改进优先级**:如果只做 3 件事来规范化这个项目,应该做哪 3 件?按优先级排序。

**输出要求**:
- 写入 `docs/audit.md`
- 标题是 "# 项目体检报告 - [日期]"
- 每个问题一个小标题
- 语言简洁,不要空话套话
- 发现问题就直说,不要委婉

**完成标志**:`docs/audit.md` 存在且内容完整。让我读一遍,我确认后再继续。

## 第 4 步:清理 .gitignore 和敏感文件

**目标**:确保不该提交的文件不会被 git 追踪,特别是敏感信息。

**AI 要做的事**:

1. 根据项目技术栈,生成一份完整的 `.gitignore`。如果已存在,只做补充,不要删原有内容。常见要忽略的:
   - 依赖文件夹(node_modules、venv、__pycache__ 等)
   - 构建产物(dist、build、*.pyc 等)
   - IDE 配置(.idea、.vscode/ 部分内容)
   - 系统文件(.DS_Store、Thumbs.db)
   - 环境变量(.env、.env.local)
   - 日志(*.log)
   - 临时文件

2. 扫描当前 git 仓库,检查**是否有已经被追踪但应该忽略的文件**(运行 `git ls-files` 对比 .gitignore)。如果有,列出清单问我怎么处理。

3. **重点检查敏感文件**:
   - 查找 `.env`、`.env.*`、`config.json`、`secrets.*`、`*.pem`、`*.key` 等
   - 用正则扫描代码里有没有疑似硬编码的 API key、密码(形如 `sk-xxx`、`password = "xxx"`、`token = "xxx"`)
   - **如果发现敏感信息被 git 追踪过**:
     - 立即停止,告警
     - 告诉我有哪些、在哪些文件里
     - 给我两个选项:a) 只是从追踪中删除并加入 gitignore(历史还在)  b) 从 git 历史中彻底抹除(需要额外操作)
     - 提醒我:任何泄露过的密钥**必须立刻去对应平台吊销重新生成**

4. 建议创建一个 `.env.example` 文件作为环境变量模板(不含真实值),让我知道项目需要哪些环境变量。

**完成标志**:
- `.gitignore` 规范完整
- `git status` 干净
- 没有敏感文件被 git 追踪
- 有 `.env.example`(如果项目有环境变量)

## 第 12 步:最终验收和提交

**目标**:确认一切正常,做一次规范化完成的提交。

**AI 要做的事**:

1. 运行最终检查:
   - `git status`:应该显示有大量新增的文档文件
   - 按照 README 跑一次项目,确认还能正常运行
   - 打开几个随机文件,确认没被意外改动
   - 检查 `docs/` 下所有文件都有内容(不是空的占位符)

2. 生成一份"规范化总结",告诉我:
   - 完成了哪些步骤
   - 跳过了哪些步骤(如果有)
   - 发现了哪些问题已修复
   - 哪些问题已记录但未修复(在 lessons.md / tech-design.md 的技术债部分)
   - 下一步建议做什么

3. 提示我做最后一次提交:
   ```
   git add .
   git commit -m "docs: complete project regularization
   
   - 建立标准文档结构
   - 补全 README 和技术文档
   - 配置代码格式化
   - 建立 AI 协作规则
   - 建立个人沉淀工作流"
   ```

4. 提醒我:
   - 把 `docs/checklist.md` 贴在显示器边上(或者打印出来)
   - 第一周每次提交都刻意对照一遍,两周后就会成习惯
   - 每次 AI 犯错或自己踩坑,立刻记 `docs/ai/lessons.md`

**完成标志**:最终 commit 完成,项目从野路子变成规范项目。

## 附录:常见问题

### Q:某一步做到一半出问题了怎么办?
A:运行 `git status` 看动了哪些文件,`git diff` 看具体改了什么。确认无误后 `git checkout .` 丢弃未提交的改动,回到上一步结束的状态。

### Q:AI 建议做的事我不确定要不要做?
A:直接说"跳过"或"先不做",AI 会继续下一步。规范化是分阶段做的,现在不做以后也能做。

### Q:某一步改完项目跑不起来了?
A:立刻运行 `git reset --hard HEAD`,回到最近一次提交。然后告诉 AI "跑不起来了,报错是 xxx",让 AI 重新来。这就是为什么每一步都要单独提交。

### Q:做完第几步就可以停了?
A:至少做到第 5 步(AI 规则),后面的可以分多周慢慢做。第 1-5 步是性价比最高的部分。

### Q:以后怎么维护?
A:每周花 10 分钟扫一遍 `docs/ai/lessons.md`,每个月看一次 `docs/audit.md` 对比现在的项目状态,看有没有进步。

---

**最后一句**:规范化不是为了好看,是为了让你以后改代码不痛苦。每一步的价值,都是**未来的你省下来的时间和焦虑**。
